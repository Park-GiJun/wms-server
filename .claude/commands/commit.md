# /commit — 커밋 + 작업 히스토리 기록

현재 작업 트리의 변경사항을 커밋하면서, 저장소 루트 `history/` 아래에 날짜별 디렉토리를 만들어
커밋 단위 작업 히스토리 문서를 함께 남긴다. **히스토리 문서 작성은 반드시 Sonnet 모델
서브에이전트에게 위임한다** (본 세션 모델이 직접 쓰지 않는다).

## 입력

- `$ARGUMENTS`: 커밋 메시지 힌트(선택). 있으면 커밋 메시지·히스토리 제목에 반영한다.

## 절차 (순서대로)

### 1. 변경사항 파악
- `git status` 와 `git diff HEAD` (스테이징 여부 무관하게 전체)를 확인한다.
- 변경이 하나도 없으면 **중단**하고 사용자에게 알린다.
- 최근 커밋 스타일 참고를 위해 `git log --oneline -5` 를 본다.

### 2. 히스토리 경로·시퀀스 결정
- 오늘 날짜를 구한다: `Get-Date -Format yyyyMMdd` (예: `20260705`).
- 디렉토리: `history/<YYYYMMDD>/` (없으면 생성).
- 그 안의 기존 파일을 Glob(`history/<YYYYMMDD>/*.md`)으로 조회해 **다음 시퀀스 번호**를 정한다.
  - 파일명 규칙: `<YYYYMMDD>-<NN>.md`, `NN` 은 `01`~`99` 2자리 zero-pad.
  - 기존 최대 번호 + 1. 없으면 `01`. `99` 를 넘으면 중단하고 사용자에게 알린다.

### 3. 히스토리 문서 작성 — **Sonnet 서브에이전트**
Agent 툴을 다음 옵션으로 호출한다 (**반드시 `model: "sonnet"`, `run_in_background: false`**):

- 프롬프트에 다음을 그대로 담아 전달한다:
  - `git diff HEAD` 전문(너무 길면 `git diff HEAD --stat` + 핵심 파일 diff 발췌)
  - `git status` 결과, 사용자 힌트(`$ARGUMENTS`), 이번 세션에서 한 작업의 요약
  - 작성 대상 파일 경로: `history/<YYYYMMDD>/<YYYYMMDD>-<NN>.md`
- 서브에이전트에게 아래 템플릿으로 **한국어** 문서를 해당 경로에 Write 하도록 지시한다:

```markdown
# <작업 한 줄 요약>

- **일시**: YYYY-MM-DD HH:mm
- **커밋**: <커밋 메시지 제목 한 줄 — 해시는 적지 않는다(커밋이 자기 해시를 담을 수 없음)>

## 작업 내용
<무엇을 왜 했는지 2~5문장>

## 변경 파일
<파일 경로 목록 + 각 파일에서 바뀐 것 한 줄>

## 주요 결정사항
<설계/구현 판단이 있었다면. 없으면 "없음">

## 후속 작업
<남은 일이 있다면. 없으면 "없음">
```

- 서브에이전트가 끝나면 파일이 실제로 생성됐는지 확인한다.

### 4. 커밋
- 히스토리 파일을 포함해 스테이징한다: `git add -A`.
  - 단, `.env`·자격증명 등 시크릿으로 보이는 파일이 diff 에 있으면 **커밋 전 중단**하고 사용자에게 확인.
- 커밋 메시지: 변경 내용 기반 한 줄 요약(힌트 반영), 마지막에
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` 추가.
- 히스토리 문서의 `**커밋**:` 에는 이 커밋 메시지 제목을 미리 적어 둔다.
  **커밋 해시는 기록하지 않는다** — amend 로 채워 넣으면 해시가 다시 바뀌어 항상 어긋난다.

### 5. 보고
- 커밋 해시, 커밋 메시지, 히스토리 파일 경로(`history/<YYYYMMDD>/<YYYYMMDD>-<NN>.md`)를 사용자에게 알린다.

## 규칙

- **push 는 하지 않는다** (사용자가 명시적으로 요청한 경우만).
- 히스토리 문서 작성 주체는 항상 Sonnet 서브에이전트 — 비용 절감 목적이므로 직접 쓰지 말 것.
- `history/` 는 git 추적 대상이다(.gitignore 에 넣지 않는다).
- 같은 날 여러 번 커밋하면 `-01`, `-02`, … 로 파일이 쌓인다.

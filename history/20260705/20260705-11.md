# user-service 를 master-service 로 통합

- **일시**: 2026-07-05 21:26

## 작업 내용
상시 기동 서비스 수를 줄이기 위해(향후 inbound/outbound 피처 추가 대비) user-service(신원 마스터 + JWT 발급, 포트 19103)를 척추인 master-service(19102)로 흡수 통합했다. 소스 전체를 `com.gijun.wms.user` → `com.gijun.wms.master` 패키지로 rename 하여 이동했고, app_user 테이블은 master 의 Flyway V2 마이그레이션으로 옮겨 wms_master DB 에 생성되도록 했다. gateway 라우트·config-repo·배포 compose·TeamCity·문서를 모두 함께 갱신했다. 전체 빌드 및 테스트(PasswordPolicyTest) 통과 확인.

## 주요 결정사항
1. 통합 방향은 user → master (master 가 척추이자 마스터 데이터 소유자). JWT 발급 책임도 master 로 이동 — "신원 마스터 소유자가 발급"이라는 원칙은 유지.
2. 패키지는 com.gijun.wms.master 로 rename 하여 컴포넌트 스캔 범위에 자연 편입 (별도 scanBasePackages 없음).
3. gateway 의 user-auth/user-core 두 라우트를 master-auth 하나(Path=/api/auth/**,/api/users/**)로 합침.
4. inventory-service 통합(76b95d5)과 동일한 선례를 따라 단일 커밋으로 처리.

## 후속 작업
홈서버에 이미 wms_user DB 데이터가 있으면 배포 전 wms_master 로 수동 이관 필요(없으면 wms_user DB 삭제). platform-server 는 config-repo 볼륨이라 재빌드 불필요하지만 gateway·master-service 는 재배포 필요.

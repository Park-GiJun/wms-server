# user-service 회원가입(승인제)·로그인/JWT 발급·Swagger 구현

- **일시**: 2026-07-05
- **커밋**: feat[user-service] : 회원가입(승인제)·로그인/JWT 발급·Swagger 구현

## 작업 내용
user-service 의 appUser 도메인을 헥사고날+CQRS 로 구현했다. 흐름은 register(PENDING) → 관리자
approve/reject(ACTIVE/REJECTED) → login(ACTIVE 만 JWT 발급) 순서다. 발급 토큰의 클레임
(subject=userId, email, role, issuer)은 shared 의 JwtTokenValidator(gateway 검증 지점)와 일치하도록
맞췄다. springdoc 기반 Swagger UI(bearerAuth 스킴)를 추가했고 전체 빌드가 통과했다.

## 주요 결정사항
- 회원가입은 별도 어그리게이트/이벤트 없이 AppUser 상태머신으로 처리(오버엔지니어링 회피).
- 로그인 실패 시 계정 존재 여부를 은닉한다. 자격 검증을 상태 검증보다 먼저 수행하고 401 단일 메시지로 응답.
- 엔티티는 불변(val)+save merge 스타일로 작성.
- Kotlin 블록 주석 중첩 함정(`/api/auth/**` 를 KDoc 에 쓰면 unclosed comment) 발견 및 회피.

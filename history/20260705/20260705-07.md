# inventory-service 를 master-service 로 통합

- **일시**: 2026-07-05
- **커밋**: refactor[master-service] : inventory-service 를 master-service 로 통합

## 작업 내용
서비스 수를 줄이기 위해 inventory-service(재고원장 척추)를 master-service 로 통합했다. 두 서비스 모두 도메인 코드가 없는 스캐폴드 단계라 코드 이동 없이 모듈 삭제 + 참조 이관으로 끝났다. master-service 가 이제 척추(재고원장·stock.movement 발행)와 품목·로케이션·거래처 마스터를 함께 소유한다(포트 19102 유지, 19101 반납). 공유 Postgres 의 빈 wms_inventory database 는 드랍했다(테이블 0개 확인 후). 전체 빌드 성공 확인.

## 변경 파일
- backend/inventory-service/** 삭제, config-repo/inventory-service.yml 삭제
- backend/master-service — build/Application/V1__init.sql 에 척추(재고원장) 역할 서술 병합
- config-repo/master-service.yml — inventory 토픽/allow-negative 설정 블록 이관
- config-repo/gateway.yml — inventory 경로(/api/inventory/**,/api/stock-movements/**)를 master-core 라우트로 통합
- backend/settings.gradle.kts, .teamcity/settings.kts — inventory 모듈/빌드타입 제거
- backend/Dockerfile — inventory COPY 제거 + 누락돼 있던 user-service COPY 추가(스캐폴드 때 빠진 버그 수정)
- deploy/docker-compose.yml — inventory 블록 제거, master 를 척추로 서술
- .env.example (+미추적 .env) — INVENTORY_DB_URL 제거
- backend/shared/StockMovementEvent.kt, backend/platform-server/PlatformServerApplication.kt — 발행 주체/기동 순서 주석 갱신
- CLAUDE.md, README.md, .claude/commands/new-service.md, .claude/commands/commit.md — 척추=master-service 로 문서 일괄 갱신

## 주요 결정사항
- 통합 방향은 master 로(inventory 를 master 에 흡수, 포트 19102 유지)
- 단일 라이터 척추 규칙은 그대로 — 이제 master-service 가 척추라 "척추는 건드리지 않는다" 규칙의 대상이 master-service 로 변경
- 빈 wms_inventory DB 는 안전확인(테이블 0개) 후 드랍

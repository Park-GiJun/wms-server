# 신원/JWT 발급을 master-service 에서 user-service 로 분리

- **일시**: 2026-07-05
- **커밋**: feat[user-service] : 신원/JWT 발급을 master-service 에서 user-service 로 분리

## 작업 내용
신원(user) 마스터와 JWT 발급 책임을 master-service 에서 분리해 새 user-service(포트 19103)를 `/new-service` 컨벤션으로 신설했다. master 에는 실제 구현은 없었고(스캐폴드 주석·의존성뿐) 의도·의존성·설정만 이관하는 작업이었다. gateway 의 인증 라우트도 신설 서비스로 이관했다.

## 변경 파일
- backend/user-service/build.gradle.kts (신규) — 표준 피처 서비스 + spring-security-crypto/jjwt(JWT 발급)
- backend/user-service/src/main/kotlin/com/gijun/wms/user/UserServiceApplication.kt (신규)
- backend/user-service/src/main/resources/application.yml (신규) — 부트스트랩(이름+config 주소)
- backend/user-service/src/main/resources/db/migration/V1__init.sql (신규) — Flyway 베이스라인(app_user 예정)
- config-repo/user-service.yml (신규) — 포트 19103, wms_user DB, jwt validity
- backend/master-service/build.gradle.kts — security-crypto/jjwt 의존성 제거
- backend/master-service/…/MasterServiceApplication.kt, V1__init.sql, config-repo/master-service.yml — 신원/JWT 서술 제거
- backend/settings.gradle.kts — user-service 등록
- config-repo/gateway.yml — /api/auth/** 라우트를 user-service 로 이관 + /api/users/** 라우트 추가
- backend/shared/build.gradle.kts, JwtTokenValidator.kt — 발급 주체 주석 master→user
- .teamcity/settings.kts — backendServices 에 user-service 추가

## 주요 결정사항
신원 소유 서비스가 JWT 발급을 담당한다(검증은 gateway, shared JwtTokenValidator 공유). gateway 의 auth 라우트가 원래 별도(master-auth)라 uri 만 교체했다.

## 후속 작업
user 도메인 실제 구현(app_user DDL, 로그인/발급 API).

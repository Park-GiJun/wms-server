# ddl-auto validate 고정 — 스키마 버전관리는 Flyway 단일 창구

- **일시**: 2026-07-05
- **커밋**: chore[config-repo] : ddl-auto validate 고정 — 스키마 버전관리는 Flyway 단일 창구

## 작업 내용
Hibernate 가 스키마를 변경할 수 있던 `ddl-auto: ${JPA_DDL_AUTO:update}` 를 제거하고 `validate` 로 하드코딩했다. 이제 스키마 버전관리는 Flyway(V__*.sql) 단일 창구가 되고, Hibernate 는 엔티티-스키마 일치 검증만 수행한다. 환경변수 노브(JPA_DDL_AUTO) 자체를 없애 실수로 `update` 로 되돌릴 수 없도록 했다.

## 변경 파일
- config-repo/application.yml — ddl-auto 를 validate 하드코딩(+주석)
- config-repo/application-prod.yml — 중복이 된 prod jpa 오버라이드 블록 제거
- .env.example — JPA_DDL_AUTO 항목 제거 (미추적 .env 에서도 제거함)

## 주요 결정사항
- ddl-auto 는 env 노브 없이 validate 고정(none 이 아닌 validate — 엔티티와 Flyway 스키마 불일치를 부팅 시점에 잡는다).

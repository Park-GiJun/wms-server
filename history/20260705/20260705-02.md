# /commit 커맨드에 push·서비스별 분리 커밋·메시지 형식 추가

- **일시**: 2026-07-05
- **커밋**: chore[claude] : /commit 커맨드에 push·서비스별 분리 커밋·메시지 형식 추가

## 작업 내용
`/commit` 슬래시 커맨드(`.claude/commands/commit.md`)를 개정했다. 개인 프로젝트 특성상 커밋 후
`git push` 를 자동으로 수행하도록 하고, 커밋 메시지를 `<type>[<scope>] : <요약>` 형식으로 통일했다.
또한 변경 파일을 서비스(모듈) 단위로 그룹핑해 그룹당 커밋 1개·히스토리 문서 1개로 분리 생성하도록
바꾸고, 히스토리 문서에는 커밋 해시 대신 커밋 메시지 제목을 기록하도록 했다.

## 변경 파일
- `.claude/commands/commit.md` (수정)

## 주요 결정사항
- 커밋 후 `git push` 를 자동 수행한다(개인 프로젝트 전제).
- 커밋 메시지 형식을 `<type>[<scope>] : <요약>` 으로 고정한다. type 은
  feat/fix/refactor/docs/chore/test/ci, scope 는 모듈명 또는
  config-repo/infra/deploy/docs/claude/build 중 하나를 쓴다.
- 변경 파일은 서비스(모듈) 단위로 그룹핑하며, 그룹 1개 = 커밋 1개 = 히스토리 문서 1개로 분리
  커밋한다. 단, 새 서비스 스캐폴드처럼 논리적으로 하나의 작업이면 여러 파일이 걸쳐 있어도 한
  그룹으로 묶는 것을 우선한다.
- 그룹별 스테이징 시 `git add -A` 는 금지하고 그룹에 속한 파일만 명시적으로 스테이징한다.
- 히스토리 문서에는 커밋 해시 대신 커밋 메시지 제목을 기록한다(커밋이 자기 해시를 담을 수
  없어 amend 루프가 발생하는 문제를 해결하기 위함).
- 그룹이 여러 개이더라도 Sonnet 에이전트 호출은 1회로 통합한다.

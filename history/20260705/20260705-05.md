# datasource·JWT fallback 제거 + 홈서버 공유 인프라 접속 체계

- **일시**: 2026-07-05
- **커밋**: chore[config] : datasource·JWT fallback 제거 + 홈서버 공유 인프라 접속 체계

## 작업 내용
로컬 dev 를 홈서버 공유 인프라(Postgres 5432 · Redis 6380 · Kafka 9094)에 직결하는 체계로 전환했다. config-repo 의 datasource H2 fallback 과 JWT_SECRET 가짜 기본값을 제거했다 — 값이 없으면 부팅 실패가 정상("정보 못찾으면 못키는거임"). 모든 서비스가 같은 repo 루트 .env 를 읽는데 DB-per-service 라 DB URL 을 서비스별 변수(MASTER_DB_URL/INVENTORY_DB_URL/USER_DB_URL)로 분리했다(계정은 DB_USERNAME/DB_PASSWORD 공유). 공유 Postgres 에 wms_master/wms_inventory/wms_user database 를 JDBC 로 생성 완료했다. H2 는 테스트 전용(testRuntimeOnly)으로 강등했다. 실제 접속값은 미추적 .env 에 작성했고 .env.example 은 구조만 담는다.

## 변경 파일
- config-repo/application.yml — JWT_SECRET fallback 제거
- config-repo/{master,inventory,user}-service.yml — datasource fallback 제거·서비스별 DB URL 변수 (master/user 는 문서 1 커밋에 포함)
- backend/{master,inventory,user}-service/build.gradle.kts — h2 를 testRuntimeOnly 로
- backend/inventory-service (application.yml 주석, Application KDoc)
- deploy/docker-compose.yml — user-service 블록 추가, DB_URL → 서비스별 변수명, master 에서 JWT env 제거
- deploy/.env.example, .env.example — 새 변수 체계 반영
- README.md, CLAUDE.md — user-service 표 반영·인프라/기동 서술 갱신
- .claude/commands/new-service.md — 템플릿을 fallback 없는 <UPPER>_DB_URL 컨벤션으로, 포트 안내 19104 부터

## 주요 결정사항
시크릿·자격증명은 fallback 금지(주소성 값만 기본값 허용). DB URL 서비스별 변수화(단일 .env 에서 DB-per-service 구분). H2 dev 폴백 폐지.

# stock.movement 첫 컨슈머(로그) 추가

- **일시**: 2026-07-12 19:39
- **커밋**: feat[notification-service] : stock.movement 첫 컨슈머(로그) 추가

## 작업 내용
재고원장 5단계 마무리로, notification-service 에 stock.movement 토픽을 구독하는 첫 컨슈머를 추가했다. 아직 알림 도메인이 설계되지 않은 단계라 우선 수신 로그만 남기는 어댑터 골격으로 파이프라인 연결을 검증했다. 이로써 커맨드→원장→발행→구독으로 이어지는 재고원장 파이프라인이 완성됐다.

## 변경 파일
- `backend/notification-service/src/main/kotlin/.../infrastructure/adapter/in/message/StockMovementEventListener.kt` — 신규. @KafkaListener 로 stock.movement 구독, 파이프라인 검증용 로그만. 실제 알림 규칙이 생기면 유스케이스 호출로 교체, 멱등 처리는 idem:{movementId}(Redis) 예정
- `config-repo/notification-service.yml` — kafka.consumer 에 String/Json deserializer + spring.json.use.type.headers=false(타입 헤더 대신 default.type 고정 역직렬화 — 서비스 간 패키지 결합 최소화) + trusted.packages

## 주요 결정사항
1. 역직렬화는 발행자 타입 헤더가 아닌 value.default.type 고정 — 컨슈머가 발행자의 클래스 경로에 결합되지 않게.
2. 첫 컨슈머는 로그만 — 알림 도메인 설계 전이라 어댑터 골격만 검증.

## 후속 작업
- 알림 규칙(안전재고 미달 등) 도메인 설계 + Redis 멱등 처리

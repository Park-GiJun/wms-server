# 재고원장 코어(StockMovement/StockBalance) + 이동 커맨드 API 구현

- **일시**: 2026-07-12 18:56
- **커밋**: feat[master-service] : 재고원장 코어(StockMovement/StockBalance) + 이동 커맨드 API 구현

## 작업 내용
wms-server 의 척추인 재고원장을 구현했다. 모든 물리 이동을 append-only stock_movement 로 환원하고, (skuId, locationId) 단위 stock_balance 가 qty >= 0 불변식을 집행하는 단일 라이터 관문이다. 도메인(2단계)과 커맨드 경로(3단계)를 함께 담았다. master-service 빌드·테스트 전체 통과.

## 변경 파일
domain:
- `domain/stock/StockMovementModel.kt` — 신규. 불변 원장 레코드(부호 있는 qty, (sku,location) 쌍 단위 seq)
- `domain/stock/StockBalanceModel.kt` — 신규. 단일 라이터 관문. apply() 하나로 qty=0 거부 → 음수 결과 차단 → seq=lastSeq+1 movement 생성 + 잔고 갱신. empty() 팩토리와 StockApplication(잔고+movement 쌍) 포함
- `domain/stock/StockLedger.kt` — 신규. 도메인 서비스(object). 이동 종류별 부호 정책: receive(양수만)/adjust(유일한 ± 양방향)/transfer(2건 분해: 출발지 -qty + 도착지 +qty, 같은 로케이션 거부, 같은 SKU 강제)
- `domain/stock/exception/StockException.kt` — 신규. sealed: InsufficientStockException(422)/InvalidQuantityException(400)/SameLocationTransferException(400)/InactiveTargetException(409)

application:
- `application/dto/command/StockCommands.kt` — RecordReceipt/RecordTransfer/RecordAdjustment 커맨드 (사용자 초안 유지)
- `application/dto/result/StockResults.kt` — 신규. StockMovementResult(movement + 적용 후 balanceQty), TransferResult(outbound/inbound)
- `application/port/in/command/RecordReceiptCommandUseCase.kt`, `RecordTransferCommandUseCase.kt`, `RecordAdjustmentCommandUseCase.kt` — 사용자 초안의 메서드명(receive/transfer/adjust) 유지, 반환 타입 추가
- `application/port/out/persistence/stock/StockCommandPersistencePort.kt` — 신규. ensureAndLock(없으면 생성 후 FOR UPDATE 락) + persist(id 채워진 movement 반환)
- `application/port/out/persistence/item/ItemQueryPersistencePort.kt` — findSkuById 시그니처 추가(어댑터에 구현 이미 존재)
- `application/handler/command/StockCommandHandler.kt` — 신규. 마스터 검증 → 락 → StockLedger → 기록 배관. transfer 는 locationId 오름차순 락(데드락 방지), 도메인엔 원래 방향 전달

## 주요 결정사항
1. 원장 코어는 domain 안(CLAUDE.md 척추 예외 규칙) — 검증·적용 규칙 전부 순수 함수, 테스트가 DB 없이 돈다.
2. TRANSFER 는 movement 2건 분해 + 락은 locationId 오름차순(데드락 방지), 도메인엔 원래 방향 전달.
3. 잔고 행 생성 경합은 INSERT-먼저(ON CONFLICT DO NOTHING) 후 FOR UPDATE 로 제거 — "SELECT 후 INSERT" 는 없는 행에 락이 안 걸림.
4. 마스터 검증 정책: 미존재 404, 유입(+) 방향만 ACTIVE 요구(비활성 대상도 반출·조정 가능), 위반 409.
5. adjust 의 reason 은 스키마에 컬럼이 없어 v1 API 에서 제외 — 실사 피처 때 V7 로 추가.
6. 사용자 초안(StockCommands.kt, 유스케이스 3개 메서드명) 존중 — 반환 타입만 추가.

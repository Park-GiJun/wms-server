# 주석 보강 슬래시 커맨드(/comment) 추가

- **일시**: 2026-07-12 15:25
- **커밋**: feat[claude] : 주석 보강 슬래시 커맨드(/comment) 추가

## 작업 내용
대상 코드에 프로젝트 컨벤션에 맞는 한국어 주석(KDoc)을 보강하는 /comment 슬래시 커맨드를 추가했다. 인자로 경로를 받고, 생략 시 현재 작업 트리의 변경 파일이 대상이다. 코드 로직은 한 글자도 바꾸지 않고 주석만 추가/개선하며, 커밋은 하지 않는다(/commit 과 분리). 기존 코드(AppUserModel, ItemCommandHandler 등)의 주석 밀도·톤을 기준으로 삼는다.

## 변경 파일
- `.claude/commands/comment.md` — 신규. /comment 커맨드 정의(입력/주석 컨벤션/절차/규칙)

## 주요 결정사항
1. 주석 컨벤션을 커맨드 안에 명문화 — 한국어, 코드가 스스로 말하지 못하는 것만(도메인 불변식/설계 의도/경계/HTTP 매핑), 라인 반복·변경 이력·리뷰어용 설명은 금지 유형.
2. 배치 기준을 4단계로: 클래스 상단 KDoc(원칙 필수) / 함수 KDoc(비자명할 때만) / 인라인 //(특정 라인 제약만) / SQL(테이블 용도·비자명 컬럼).
3. "과잉 주석이 누락보다 나쁘다" — 판단이 서지 않으면 달지 않는 것이 기본값. 이미 충분한 코드는 건드리지 않음.
4. 주석 외 diff(import·공백·포매팅) 금지, 테스트 파일은 클래스 상단 한 줄 외 금지.

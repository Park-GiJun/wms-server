# 출고(outbound) 피처 서비스 스캐폴드

- **일시**: 2026-07-12 17:07
- **커밋**: feat[outbound-service] : 출고 피처 서비스 스캐폴드

## 작업 내용
`/new-service outbound` 로 출고 피처 서비스 골격을 생성했다. 포트는 inbound-service 다음인 19106,
패키지는 `com.gijun.wms.outbound` 로 산정했다. 주문·피킹·패킹·출고(PICK/PACK/SHIP) 커맨드를
척추(master-service)로 보내는 서비스가 될 예정이며, 구성 패턴은 inbound-service 와 동일하게
맞췄다. 이번 범위는 도메인 코드 없이 빌드/실행 골격만 채웠다.
`.\gradlew.bat :outbound-service:build` 빌드 검증을 통과했다.

## 변경 파일
- `backend/outbound-service/build.gradle.kts` — 신규. JPA+Kafka+Redis+Flyway 의존성, shared 모듈 참조
- `backend/outbound-service/src/main/kotlin/com/gijun/wms/outbound/OutboundServiceApplication.kt` — 신규. 부트 진입점
- `backend/outbound-service/src/main/resources/application.yml` — 신규. 식별자+Config 서버 주소만
- `backend/outbound-service/src/main/resources/db/migration/V1__init.sql` — 신규. Flyway 베이스라인
- `config-repo/outbound-service.yml` — 신규. 포트 19106, OUTBOUND_DB_URL(fallback 없음), Kafka 그룹 wms-outbound
- `backend/settings.gradle.kts` — include("outbound-service") 추가(주석 자리 교체)
- `config-repo/gateway.yml` — outbound-core 라우트(Path=/api/outbound/**) 추가

## 주요 결정사항
1. 포트는 19106(inbound-service 19105 다음)으로 산정.
2. 나머지 스캐폴드 규칙(의존성 조합, 도메인 코드 제외, JWT 발급 미포함 등)은 inbound-service 와 동일하게 적용.

## 후속 작업
- 공유 Postgres 에 `wms_outbound` DB 생성 + `.env` 에 `OUTBOUND_DB_URL` 추가해야 부팅 가능.
- `deploy/docker-compose.yml` 항목 추가 및 `.teamcity/settings.kts` 의 `backendServices` 등록.
- 도메인 코드(주문·피킹·패킹·출고)는 재고원장(master-service) 구현 후 착수.

# 재고 잔고·원장 이력 조회 API 구현

- **일시**: 2026-07-12 19:13
- **커밋**: feat[master-service] : 재고 잔고·원장 이력 조회 API 구현

## 작업 내용
재고원장 4단계(조회 경로). 직전 커밋의 커맨드 경로에 이어 CQRS 의 조회쪽을 붙였다. GET /api/inventory/balances (현재 잔고, skuId/locationId 필터 선택) 와 GET /api/stock-movements (원장 이력 최신순, 필터 하나 이상 필수 — append-only 라 무한히 자라는 이력의 무필터 전체 덤프를 400 으로 거부). gateway 라우트는 둘 다 master-core 에 이미 있어 변경 없음. master-service 빌드·테스트 전체 통과.

- `domain/stock/exception/StockException.kt` — EmptyMovementFilterException(400) 추가

- `infrastructure/adapter/out/persistence/stock/StockPersistenceAdapter.kt` — StockQueryPersistencePort 구현 추가(when 조합, 기존 슬라이스 패턴)
- `infrastructure/adapter/in/stock/web/InventoryController.kt` — 신규. GET /api/inventory/balances

- `src/test/kotlin/.../application/handler/query/StockQueryHandlerTest.kt` — 신규. 3케이스(무필터 이력 거부, 필터 시 조회, 잔고 무필터 허용)

## 주요 결정사항
1. 원장 이력은 skuId/locationId 중 하나 이상 필수 — append-only 테이블의 무필터 전체 덤프 방지(400).
2. 잔고 조회는 무필터 허용 — (sku × location) 조합으로 상한이 있는 파생 상태라서.
3. 이력 정렬은 id 내림차순 — seq 는 (sku, location) 쌍 안에서만 의미가 있어 교차 필터 조회의 전역 정렬 키로 부적합.
4. 페이징은 이번 범위 제외 — 기존 마스터 목록 API 와 동일하게 전체 반환, 필요 시 후속.

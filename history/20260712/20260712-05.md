# 입고(inbound) 피처 서비스 스캐폴드

- **일시**: 2026-07-12 17:07
- **커밋**: feat[inbound-service] : 입고 피처 서비스 스캐폴드

## 작업 내용
`/new-service inbound` 로 입고 피처 서비스 골격을 생성했다. 포트는 191xx 대역의 다음 빈 포트인
19105, 패키지는 `com.gijun.wms.inbound` 로 산정했다. 입고지시·검수 후 RECEIPT/PUTAWAY 커맨드를
척추(master-service)로 보내는 서비스가 될 예정이며, 이번 범위는 도메인 코드 없이 빌드/실행 골격만
채웠다. 의존성 조합은 notification-service 와 동일하게 JPA+Kafka+Redis+Flyway 로 맞췄다.
`.\gradlew.bat :inbound-service:build` 빌드 검증을 통과했다.

## 변경 파일
- `backend/inbound-service/build.gradle.kts` — 신규. JPA+Kafka+Redis+Flyway 의존성, shared 모듈 참조
- `backend/inbound-service/src/main/kotlin/com/gijun/wms/inbound/InboundServiceApplication.kt` — 신규. 부트 진입점(@ConfigurationPropertiesScan)
- `backend/inbound-service/src/main/resources/application.yml` — 신규. 식별자+Config 서버 주소만(실설정은 config-repo)
- `backend/inbound-service/src/main/resources/db/migration/V1__init.sql` — 신규. Flyway 베이스라인(SELECT 1)
- `config-repo/inbound-service.yml` — 신규. 포트 19105, INBOUND_DB_URL(fallback 없음), Kafka 그룹 wms-inbound
- `backend/settings.gradle.kts` — include("inbound-service") 추가(주석 자리 교체)
- `config-repo/gateway.yml` — inbound-core 라우트(Path=/api/inbound/**) 추가

## 주요 결정사항
1. 포트는 config-repo 의 `server.port` 최대값+1 규칙으로 19105 산정.
2. 도메인 코드는 포함하지 않고 골격만 스캐폴드(확장 규칙: 척추는 건드리지 않는다).
3. JWT 발급 의존성은 미포함(발급은 master-service 단일 지점 원칙 유지).

## 후속 작업
- 공유 Postgres 에 `wms_inbound` DB 생성 + `.env` 에 `INBOUND_DB_URL` 추가해야 부팅 가능.
- `deploy/docker-compose.yml` 항목 추가 및 `.teamcity/settings.kts` 의 `backendServices` 등록.
- 도메인 코드(입고지시·검수)는 재고원장(master-service) 구현 후 착수.
